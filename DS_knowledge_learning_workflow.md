# DS / 统计 / 编程 / ML 知识学习 Workflow

## 核心目标

不是“学得很满”，而是：

> 课前不迷路，课中抓重点，课后能沉淀，作业能反推知识。

---

## 1. 总流程图

```text
拿到 slides / reading
↓
课前：轻预习，做“听课导航”
↓
上课：不求全懂，只抓重点和半懂
↓
课后当天：10–20分钟快速回收
↓
作业前：把作业和 lecture concept 对上
↓
作业中：GPT / Codex 辅助，但不直接外包大脑
↓
作业后：反向总结，形成自己的知识卡
↓
每周：压缩复盘，整理成可考试/可项目调用的结构
```

---

## 2. 课前 Workflow：轻预习，不要深度逐页死磕

### 课前目标

课前不是为了全部学会，而是为了：

```text
1. 知道这节课讲什么
2. 知道老师讲到每页时该看哪里
3. 知道哪些地方我大概率会卡
4. 带着问题去听课
```

---

### Step 1：先让 GPT 做整体地图

不要问：

```text
请逐页详细讲解。
```

改成：

```text
这是我的 lecture slides。请不要逐页详细讲解。

请帮我做课前预习地图：
1. 把这节课分成 3–5 个模块
2. 每个模块用一句大白话解释
3. 标出最重要的 5 个关键词
4. 标出最值得预习的 3–5 页
5. 告诉我这节课可能和作业/考试怎么关联
6. 给我 3 个上课时要重点听的问题
```

---

### Step 2：逐页轻扫，不逐页深学

每一页最多 1–2 分钟。

对每一页问 GPT：

```text
请帮我做这一页的“听课路标”，不要详细展开：

1. 这一页一句话在讲什么？
2. 我上课时眼睛应该先看哪里？
3. 老师讲的时候我要抓哪 3 个关键词？
4. 这页最低掌握标准是什么？
5. 哪些细节可以先放过？
6. 我最可能卡在哪里？
```

### 这一页的听课路标模板

```text
Page X

一句话：

关键词：

图/公式先看哪里：

我可能会卡的地方：

这页最低掌握标准：
```

---

### Step 3：做课前问题清单

预习结束后，只保留 3–5 个问题。

```text
我上课要重点听：
1. 老师怎么解释 ______？
2. ______ 为什么要这样做？
3. ______ 和作业有什么关系？
4. 这个公式/图到底在表达什么？
5. 老师有没有说 common mistake？
```

### 课前时间限制

```text
一节课 slides 预习总时间：30–60 分钟
不要超过 1 小时
```

如果超过 1 小时，很容易课前耗尽脑力，上课反而崩。

## 3. 上课 Workflow：不求全懂，只抓三类东西

上课时不要试图同时做到：

```text
听懂老师
看懂图
理解公式
记完整笔记
思考作业
```

这会爆炸。

你上课只做三件事：

```text
⭐ 老师强调
⚠️ 半懂
A 作业相关
```

---

### 1. ⭐ 老师强调

只要老师说这些话，就标 ⭐：

```text
This is important.
You need to know this.
Common mistake.
In the assignment...
In practice...
The key idea is...
Remember that...
```

记录关键词即可：

```text
⭐ Bagging reduces variance
⭐ Decision trees are unstable
⭐ Bootstrap = sampling with replacement
```

---

### 2. ⚠️ 半懂停车场

上课听到半懂，不要当场死磕。直接写进停车场：

```text
⚠️ 为什么有放回抽样？
⚠️ argmax 公式怎么读？
⚠️ variance 为什么会下降？
```

半懂不是失败。半懂只要被抓住，就没有被浪费。

---

### 3. A 作业相关

凡是老师提到这些，都标 A：

```text
dataset
code
input
output
evaluation metric
report
assignment
exam
implementation
interpretation
```

例子：

```text
A 作业可能要比较 single tree vs bagging
A 可能要解释 overfitting
A 可能要写 model evaluation
```

---

## 4. 课后当天 Workflow：10–20分钟回收

这是最重要的一步。

### 课后 5 句话总结

```text
1. 今天这节课大概讲：
2. 它解决的问题是：
3. 老师最强调：
4. 我最不懂：
5. 作业可能会用在：
```

例子：

```text
1. 今天讲 Bagging。
2. 它解决的问题是：单个模型，尤其是 decision tree，容易不稳定。
3. 老师最强调：bootstrap + 多模型投票/平均。
4. 我最不懂：为什么这样可以降低 variance。
5. 作业可能会用在：比较 single tree 和 ensemble model 的表现。
```

目标是防止这节课从脑子里蒸发。

---

### 课后只补 3 个点

不要补全世界。每节课后只补：

```text
1 个核心概念
1 个半懂问题
1 个作业相关点
```

例子：

```text
核心概念：Bagging = bootstrap + aggregation
半懂问题：为什么降低 variance？
作业相关：怎么解释 bagging model 的预测结果？
```

## 5. 作业前 Workflow：先定位知识点，再做题

作业前先做 10–15 分钟定位。

### 作业前问自己

```text
1. 这次作业对应哪几节 lecture？
2. 每题考的是哪个 concept？
3. 输入是什么？
4. 输出是什么？
5. 老师希望我展示什么能力？
```

---

### 作业前问 GPT

```text
这是我的作业题目/要求。

请先不要直接给答案。

请帮我分析：
1. 每一题对应哪一个 lecture concept
2. 这题想考我什么能力
3. 我应该回去看哪几页 slides
4. 解题的大致步骤
5. 哪些地方最容易犯错
```

---

## 6. 作业中 Workflow：GPT 和 Codex 怎么用

### GPT 适合做什么

```text
概念解释
选择题理解
统计逻辑
题意翻译
报告结构
结果解释
debug 思路
```

推荐问法：

```text
这题考的概念是什么？
请一步一步引导我想。
先不要给最终答案。
```

或者：

```text
我这样理解对吗？
如果不对，请指出我的误区。
```

---

### Codex 适合做什么

```text
读写文件
处理 input/output
写项目结构
跑代码
debug
重构代码
自动化重复任务
```

Codex 写完代码后，一定要让它反向解释：

```text
请解释这段代码的 input → process → output：

1. 输入数据是什么？
2. 每一步在处理什么？
3. 输出结果是什么？
4. 哪些地方最容易出 bug？
5. 如果我要自己重写，最小版本应该怎么写？
6. 这段代码对应哪个 lecture concept？
```

否则项目做完了，代码不一定是你的。

---

## 7. 作业后 Workflow：反向总结，真正进脑

作业交完，不要立刻结束。做 10 分钟总结。

```text
这次作业真正用到的知识：

1. Concept:
   用在哪里：

2. Concept:
   用在哪里：

3. Concept:
   用在哪里：

我一开始卡在哪里：

我后来怎么解决：

下次遇到类似题，我第一步应该做：
```

例子：

```text
这次作业用到：
1. Bootstrap
   用在：生成多个 training subsets

2. Ensemble prediction
   用在：多个模型投票/平均

3. Model evaluation
   用在：比较 bagging 和 single tree 的 accuracy

我一开始卡在：
不知道为什么要 with replacement。

下次第一步：
先画 input → model → output 流程图。
```

每周不要重新看所有 slides，只做一次压缩复盘。

### 每周复盘模板

```text
本周学了哪些主题？
1.
2.
3.

每个主题一句话：
1.
2.
3.

本周最重要的 5 个关键词：
1.
2.
3.
4.
5.

1.
2.
3.

本周作业/代码用到的知识：
1.
2.
3.

```

### 每周让 GPT 帮你整理

```text

```

## 9. 考试前 Workflow：从“看 slides”改成“做调用训练”

考试前不要只重看 slides。

考试真正考的是：你能不能把知识从脑子里拿出来用。

```text
1. 看每周知识地图
2. 每个主题用一句话解释
3. 做题
4. 错题反推概念
5. 只回看相关 slides
```

---

### 每个概念都过这 5 个问题

```text
1. 它是什么？
2. 它解决什么问题？
3. 它怎么做？
4. 它什么时候用？
5. 它容易和什么混淆？
```

例子：Bagging

```text
1. 它是什么？
一种 ensemble method。

2. 它解决什么问题？
减少单个模型不稳定。

3. 它怎么做？
bootstrap 抽样，训练多个模型，最后投票/平均。

4. 它什么时候用？
模型 variance 高，比如 decision tree。

5. 它容易和什么混淆？
Boosting / Random Forest。
```

## 10. 时间分配建议

你原来的模式可能是：

```text
课前 70%
课中 20%
课后 10%
```

建议改成：

```text
课前 30%
课中 30%
课后 40%
```

真正保存知识的动作，要放在课后和作业后。

## 11. 每日执行版

### 上课前

```text
30–60 分钟

1. GPT 总览 slides
2. 每页做听课路标
3. 写 3–5 个上课问题
```

### 上课中

```text
只记三类：

⭐ 老师强调
⚠️ 半懂
A 作业相关
```

### 下课后

```text
10–20 分钟

写 5 句话：
1. 今天讲什么
2. 解决什么问题
3. 老师强调什么
4. 我最不懂什么
5. 作业可能怎么用
```

### 做作业前

```text
10–15 分钟

先问：
1. 对应哪节课？
2. 考哪个概念？
3. 输入是什么？
4. 输出是什么？
5. 要解释什么？
```

### 做作业中

```text
GPT：理解题、概念、解释、报告
Codex：代码、文件、debug、项目结构
```

但都要加一句：

```text
请解释这对应哪个 lecture concept。
```

### 做作业后

```text
10 分钟

写：
1. 用到哪些知识
2. 我卡在哪里
3. 下次遇到类似题第一步做什么
```

## 12. 最终一句话总结

```text
课前做导航，不做重学；
课中抓重点，不求全懂；
课后做回收，不让蒸发；
作业先定位，不直接求答案；
作业后反推，把题变成知识。
```

以后不要问自己：

> 我有没有把 slides 全部看懂？

而是问：

> 这节课我能不能留下一个可调用的结构？

每节课只要留下：

```text
主题
核心问题
关键方法
一个例子
一个疑问
作业关联
```

这门课就会开始真正进脑。
