# Career Bridge

Use this reference whenever the skill produces an after-class card, assignment reverse card, weekly review, project summary, README language, or interview preparation.

## User Positioning

The user's career direction is:

```text
Applied AI / AI-native Data Science
```

Use this English positioning when useful:

```text
Data Science and Economics student focused on applied AI, agentic data workflows, and machine learning products for business, market, and decision-making problems.
```

The user is not trying to become a pure reporting analyst or a low-level dashboard operator. The learning system should help them build evidence for:

- AI-assisted coding ownership
- Data Science and ML understanding
- RAG and agent workflow awareness
- Lightweight data/AI product building
- Business, macro, risk, and decision-making interpretation

## Career Bridge Output

For each useful concept or assignment, generate:

```text
Applied AI / DS ability:
Project connection:
Portfolio bullet:
Interview sentence:
```

Keep it short. If the connection is weak, say so instead of forcing it.

## Project Lanes

### AI Data Analyst Agent

Connect concepts involving:

- EDA
- visualization
- automated summary
- business questions
- data quality checks
- human-in-the-loop validation
- report generation
- analyst workflow automation

Example portfolio language:

```text
Built an AI-assisted data analysis workflow that converts raw CSV inputs into interpretable summaries, visualizations, and human-reviewed business questions.
```

### RAG Macro Assistant

Connect concepts involving:

- document parsing
- retrieval
- embeddings
- citations
- macroeconomic indicators
- inflation, rates, FX, employment
- source-grounded answers
- uncertainty and evidence checking

Example portfolio language:

```text
Developed a source-grounded macro research assistant that retrieves economic reports and generates cited briefs on inflation, interest rates, and currency movements.
```

### Credit Risk Explainability App

Connect concepts involving:

- classification
- logistic regression
- random forest
- probability thresholds
- false positive / false negative tradeoffs
- feature importance
- SHAP
- model evaluation
- decision risk

Example portfolio language:

```text
Built an explainable credit risk app that predicts default probability, visualizes key risk drivers, and supports threshold-based decision analysis.
```

## Interview Framing

When the user used AI tools, frame it as ownership:

```text
I used AI coding tools to accelerate implementation, but I reviewed the logic, validated outputs, interpreted the results, and focused on problem framing, evaluation, and communication.
```

When connecting a course concept to work:

```text
This concept helped me understand how to turn a statistical or ML method into a decision-support workflow, not just a notebook result.
```

When explaining project readiness:

```text
I can explain the data, method, evaluation, limitations, and how I would improve the system in a real applied AI setting.
```
